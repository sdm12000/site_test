# 컨셉 분석 스킬

사용자의 웹사이트 컨셉을 전문적인 디자인 브리프로 변환한다.

## 워크플로우

### Step 1: 컨셉 파싱

사용자 프롬프트에서 다음 요소를 추출한다:

| 요소 | 추출 방법 | 기본값 (언급 없을 때) |
|------|----------|---------------------|
| 사이트 유형 | "포트폴리오", "랜딩페이지", "쇼핑몰" 등 직접 언급 | 단일 페이지 랜딩 |
| 업종/도메인 | 키워드에서 추론 | 테크/스타트업 |
| 타겟 오디언스 | "MZ세대", "기업 고객" 등 | 20-40대 디지털 네이티브 |
| 무드/분위기 | "고급스러운", "재미있는", "미니멀한" 등 | 모던 & 프리미엄 |
| 페이지 수 | 구체적 숫자 또는 "원페이지" 등 | 단일 페이지 (5~8 섹션) |
| 특별 요구사항 | "다크모드", "애니메이션 많이" 등 | 없음 |

### Step 2: 트렌드 리서치

WebSearch로 다음을 조사한다:

1. **"[업종] website design 2025 2026 trends"** — 해당 업종의 최신 디자인 트렌드
2. **"[업종] best website design awwwards"** — 해당 업종 수상작 레퍼런스
3. **"[무드 키워드] web design inspiration"** — 무드에 맞는 디자인 영감

리서치 결과에서 다음을 추출한다:
- 해당 업종에서 유행하는 레이아웃 패턴
- 인기 있는 컬러 조합
- 주목할 만한 인터랙션 패턴
- 레퍼런스 사이트 3~5개 (URL + 참고 포인트)

### Step 3: 차별화 전략 수립

AI가 만든 것 같은 뻔한 구성을 피하기 위한 차별화 포인트를 정의한다:

**피해야 할 패턴:**
- 거대 히어로 텍스트 + 중앙 CTA 버튼으로 시작하는 구성
- 3개의 동일한 피처 카드
- "우리 서비스의 장점" 섹션
- 일러스트 + 설명 텍스트 반복
- 네온 컬러 그라디언트 배경

**추구할 패턴:**
- 스토리텔링 기반 스크롤 경험
- 비정형 그리드로 시선 유도
- 텍스트와 비주얼 요소의 창의적 조합
- 인터랙티브 요소로 사용자 참여 유도
- 콘텐츠 성격에 따른 섹션별 분위기 전환

### Step 4: 브리프 작성

`_workspace/01_concept_brief.md`에 다음 구조로 작성한다:

```markdown
# 컨셉 브리프: [사이트명/컨셉명]

## 1. 프로젝트 개요
- **사이트 목적**: [무엇을 위한 사이트인지]
- **타겟 오디언스**: [누구를 위한 것인지 + 그들의 기대]
- **핵심 메시지**: [사이트가 전달해야 할 핵심 가치]
- **차별화 포인트**: [이 사이트만의 독특한 접근]

## 2. 비주얼 디렉션
- **무드 키워드**: [3~5개, 구체적으로]
- **컬러 방향**: [대략적 팔레트 방향 + 이유]
- **타이포그래피 방향**: [폰트 스타일 방향 + 이유]
- **레이아웃 스타일**: [그리드 접근법 + 이유]
- **애니메이션 톤**: [모션 성격 + 이유]
- **피해야 할 것**: [구체적으로 어떤 표현을 피할지]

## 3. 레퍼런스 사이트
| 사이트 | 참고 포인트 | URL |
|--------|-----------|-----|
| [이름] | [무엇을 참고] | [URL] |

## 4. 페이지 & 섹션 구성
### 페이지: [페이지명]
| 순서 | 섹션명 | 역할 | 콘텐츠 | 특수 요구사항 |
|------|--------|------|--------|-------------|
| 1 | [이름] | [역할] | [내용] | [애니메이션/인터랙션] |

## 5. 인터랙션 요구사항
- **페이지 로드**: [첫 인상을 위한 인트로 애니메이션]
- **스크롤**: [스크롤 기반 효과]
- **호버**: [주요 요소의 호버 반응]
- **전환**: [페이지/섹션 간 전환]
- **커서**: [커스텀 커서 필요 여부]

## 6. 기술적 고려사항
- **반응형**: [전략 — 모바일 퍼스트 vs 데스크탑 퍼스트]
- **성능**: [LCP 목표, 애니메이션 프레임레이트]
- **SEO**: [메타데이터, 구조화된 데이터 필요 여부]
- **접근성**: [최소 대비 비율, 키보드 네비게이션]
```

## 품질 기준

좋은 브리프의 조건:
- design-architect가 추가 질문 없이 디자인 시스템 작업을 시작할 수 있는 수준의 구체성
- "왜 이 방향인지" 근거가 모든 결정에 포함
- 피해야 할 것이 추구할 것만큼 명확하게 정의됨
- 레퍼런스가 실존 사이트이고, 참고 포인트가 구체적 (단순 "디자인 좋음" 불가)
