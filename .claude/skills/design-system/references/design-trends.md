## Bento 그리드

Apple이 대중화한 패턴. 다양한 크기의 카드를 그리드에 배치하여 정보 밀도와 시각적 흥미를 동시에 달성한다.

**구현 방법:**
```css
.bento-grid {
  display: grid;
  grid-template-columns: repeat(4, 1fr);
  grid-auto-rows: minmax(200px, auto);
  gap: 1rem;
}
.bento-item-large {
  grid-column: span 2;
  grid-row: span 2;
}
```

**핵심 포인트:**
- 모든 카드가 같은 크기이면 Bento가 아니라 그냥 그리드다
- 큰 카드 1~2개가 시선을 끌고, 작은 카드가 보충 정보를 제공한다
- 모바일에서는 단일 컬럼으로 자연스럽게 스택한다
- 카드 내부에 다양한 콘텐츠 타입(텍스트, 이미지, 아이콘, 차트)을 혼합한다

## Glassmorphism 2.0

2024-2025년 버전은 더 미묘하다. 과한 블러와 형광 보더 대신 자연스러운 반투명을 추구한다.

**구현 방법:**
```css
.glass-card {
  background: rgba(255, 255, 255, 0.05);
  backdrop-filter: blur(20px) saturate(1.2);
  border: 1px solid rgba(255, 255, 255, 0.08);
  border-radius: 1.5rem;
}
```

**핵심 포인트:**
- 배경에 그라디언트나 이미지가 있을 때만 효과적 — 단색 배경에는 의미 없음
- 보더 opacity를 아주 낮게 (0.05~0.1) — 과하면 촌스러워짐
- 내부 그림자(`inset box-shadow`)를 추가하면 더 자연스러운 유리 느낌

## 그라디언트 메시

유기적이고 부드러운 색상 전환. CSS conic-gradient + radial-gradient의 조합 또는 SVG 메시로 구현한다.

**구현 방법 (CSS):**
```css
.mesh-gradient {
  background:
    radial-gradient(at 40% 20%, hsla(280, 80%, 60%, 0.3) 0px, transparent 50%),
    radial-gradient(at 80% 0%, hsla(190, 80%, 60%, 0.2) 0px, transparent 50%),
    radial-gradient(at 0% 50%, hsla(340, 80%, 60%, 0.2) 0px, transparent 50%),
    radial-gradient(at 80% 80%, hsla(240, 80%, 60%, 0.15) 0px, transparent 50%);
}
```

**핵심 포인트:**
- 3~4개의 radial-gradient를 겹쳐 자연스러운 메시 효과
- 색상 간 채도와 명도 차이를 크게 두지 않아야 조화로움
- 애니메이션으로 gradient 위치를 느리게 변하게 하면 생동감 추가

## 키네틱 타이포그래피

텍스트 자체가 움직이며 메시지를 전달한다.

**유형:**
- **글자별 순차 등장**: GSAP SplitText로 글자를 분리하고 stagger 애니메이션
- **스크롤 연동 변형**: 스크롤에 따라 font-weight, size, letter-spacing이 변화
- **마퀴 텍스트**: 무한 수평 스크롤 텍스트 — 장식적 요소로 활용
- **텍스트 마스크**: 텍스트를 마스크로 사용하여 배경 이미지/비디오를 노출

## 다크모드 디자인

2025-2026의 트렌드는 순수 검정(`#000`)이 아닌 농도 있는 다크 팔레트다.

**핵심 포인트:**
- 배경: `hsl(220, 20%, 8%)` — 미세한 청색/남색 톤이 깊이감을 줌
- 카드/표면: 배경보다 3~5% 밝게 — 미묘한 레이어 구분
- 텍스트: `hsl(220, 15%, 90%)` — 순백이 아닌 미세하게 따뜻한/차가운 톤
- 액센트: 고채도 컬러가 다크 배경에서 극적으로 빛남
- 그림자: 다크모드에서는 그림자 대신 미묘한 보더나 brightness 차이로 계층 표현

## 오버사이즈 타이포그래피

히어로 섹션에서 100px+ 크기의 텍스트를 사용하여 강렬한 첫 인상을 준다.

**핵심 포인트:**
- `clamp(3rem, 10vw, 10rem)` 수준의 유동 크기
- letter-spacing을 `-.02em~-.04em`으로 타이트하게
- font-weight를 극단적으로: 100(ultra thin) 또는 900(ultra bold)
- 텍스트 자체가 디자인 요소 — 이미지 없이 텍스트만으로 히어로 구성 가능

## 노이즈/그레인 텍스처

디지털 완벽함에 아날로그 감성을 더한다.

**구현 방법 (CSS):**
```css
.grain-overlay::after {
  content: '';
  position: fixed;
  inset: 0;
  background-image: url("data:image/svg+xml,..."); /* 노이즈 SVG */
  opacity: 0.03;
  pointer-events: none;
  z-index: 9999;
}
```

또는 SVG `<feTurbulence>` 필터 사용.

**핵심 포인트:**
- opacity는 0.02~0.05 — 의식적으로 보이면 과함
- 전체 페이지에 깔거나, 특정 섹션/카드에만 적용
- 다크모드에서 더 효과적 — 어두운 배경의 미세한 질감

## 스크롤 기반 스토리텔링

스크롤을 콘텐츠 탐색이 아닌 서사 진행의 메커니즘으로 사용한다.

**유형:**
- **패럴랙스**: 전경/배경이 다른 속도로 스크롤
- **핀 섹션**: 특정 섹션이 고정된 채 내부 콘텐츠가 변화
- **프로그레스 기반**: 스크롤 진행도에 따라 애니메이션 재생
- **수평 스크롤**: 수직 스크롤을 수평 이동으로 변환

**구현:**
GSAP ScrollTrigger가 가장 안정적. Framer Motion의 `useScroll` + `useTransform`도 간단한 경우 적합.

## 3D 요소 통합

Three.js / React Three Fiber로 3D 요소를 2D 레이아웃에 통합한다.

**적합한 경우만 사용:**
- 제품 시각화 (3D 제품 뷰어)
- 배경 장식 (추상적 3D 오브젝트)
- 인터랙티브 경험 (마우스/스크롤 반응 3D)

**주의:** 3D는 성능 비용이 크다. 반드시 필요한 경우만, 모바일에서는 정적 이미지로 폴백.

## 마이크로 인터랙션

작은 인터랙션이 프리미엄 느낌을 만든다.

**필수 마이크로 인터랙션:**
- 버튼 호버: 배경색 전환 + 미세한 scale + 커서 변화
- 링크 호버: underline 드로잉 애니메이션
- 카드 호버: 그림자 깊어짐 + y축 미세 이동 + 내부 요소 미세 변화
- 입력 필드 포커스: 보더 컬러 전환 + 라벨 애니메이션
- 스크롤 인디케이터: 현재 섹션 표시 + 부드러운 전환
- 로딩 상태: 스켈레톤 또는 부드러운 펄스 애니메이션
- 페이지 전환: fade + 미세한 슬라이드

## Variable Fonts 활용

하나의 폰트 파일로 weight, width, slant 등을 연속적으로 조절한다.

**효과:**
- 호버 시 font-weight를 300→700으로 부드럽게 전환
- 스크롤에 따라 font-width가 변화
- 성능 이점: 여러 웨이트를 한 파일로 로드

**추천 Variable Fonts:**
- Inter (가변 weight)
- Clash Display (가변 weight)
- Cabinet Grotesk (가변 weight)
- Satoshi (가변 weight)

## 커서 인터랙션

커스텀 커서로 사이트의 개성을 표현한다.

**유형:**
- **커스텀 커서**: 기본 화살표 대신 커스텀 디자인의 원형/도트 커서
- **마그네틱 효과**: 버튼에 근접하면 커서와 버튼이 서로 끌림
- **팔로우 커서**: 커서 위치를 부드럽게 따라오는 원형 요소
- **컨텍스트 커서**: 호버 대상에 따라 커서 모양 변화 (이미지 → "View", 링크 → 화살표)

**주의:** 터치 디바이스에서는 비활성화하라. 접근성을 위해 `cursor: pointer` 등 기본 커서도 유지.
