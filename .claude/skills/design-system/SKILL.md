# 디자인 시스템 설계 스킬

Awwwards/FWA 수상작 수준의 디자인 시스템을 설계한다. 핵심 가치: **아름다움 > 편의성 > 관례**.

## 디자인 철학: "인간의 손길"

AI가 만든 디자인과 인간이 만든 디자인의 근본적 차이는 **의도적 불완전함**과 **맥락적 변주**에 있다. 이 디자인 시스템은 규칙 안에서의 자유로움을 추구한다.

### 핵심 원칙

1. **비대칭적 균형** — 좌우 대칭이 아닌, 시각적 무게감의 균형을 추구한다. 큰 이미지 왼쪽에 작은 텍스트를 배치하면 비대칭이지만 균형 잡힌 구도가 된다.

2. **리드미컬 스페이싱** — 모든 간격이 동일하면 단조롭다. 음악의 리듬처럼 긴장(좁은 간격)과 이완(넓은 간격)이 교차해야 한다. 관련 요소끼리는 가깝게, 그룹 간에는 멀게.

3. **레이어드 깊이감** — 평면적이지 않은, 겹침과 그림자로 깊이를 표현한다. 요소가 살짝 겹치거나, 텍스트가 이미지 위에 오거나, 부유하는 느낌의 카드를 사용한다.

4. **미세한 질감** — 순수한 flat보다 미세한 노이즈 텍스처, 그레인, 미묘한 그라디언트가 고급스럽다.

5. **의도적 대비** — 크기, 무게, 색상, 형태에서 강한 대비가 시각적 흥미를 만든다. 아주 큰 글씨 옆에 아주 작은 글씨, 둥근 요소 옆에 각진 요소.

## 컬러 시스템 설계

### 팔레트 구성 방법

1. **브랜드 컬러 1개를 기점으로 시작한다** — 컨셉 브리프의 분위기에 맞는 HSL 값을 정한다
2. **60-30-10 규칙을 적용한다** — 60% 배경, 30% 보조, 10% 강조
3. **HSL로 변형을 만든다** — 같은 Hue에서 Saturation과 Lightness를 조절하여 팔레트를 확장한다
4. **시맨틱 컬러를 정의한다** — background, foreground, muted, accent, destructive

```
@theme {
  --color-background: oklch(0.98 0.01 240);      /* 미세한 색조가 있는 배경 */
  --color-foreground: oklch(0.15 0.02 240);       /* 순수 검정이 아닌 */
  --color-muted: oklch(0.93 0.01 240);
  --color-muted-foreground: oklch(0.45 0.02 240);
  --color-accent: oklch(0.65 0.25 280);           /* 채도 높은 강조색 */
  --color-accent-foreground: oklch(0.98 0.01 280);
}
```

### 피해야 할 컬러 패턴

- 순수한 `#000000` 검정과 `#FFFFFF` 흰색 — 자연에 순수한 흑백은 없다. 미세한 색조를 더한다
- 파랑→보라 그라디언트 — AI의 기본 그라디언트. 대신 브랜드 컬러 기반 미묘한 그라디언트를 사용한다
- 네온 컬러 남용 — 포인트로만 극소량 사용한다
- 회색 일색의 뉴트럴 — 뉴트럴에도 미세한 warm/cool 톤을 준다

## 타이포그래피 시스템

### 폰트 선택 원칙

- **Display**: 개성 있는 폰트. Playfair Display, Clash Display, Cabinet Grotesk, Satoshi 등 — 해당 컨셉의 무드를 강하게 전달한다
- **Body**: 가독성 우선. Inter, Plus Jakarta Sans, Geist, Pretendard 등
- **한국어**: Pretendard, SUIT, Wanted Sans 등 웹 최적화된 한국어 폰트 병행

### Type Scale (Fluid Typography)

clamp()를 사용하여 뷰포트에 따라 자연스럽게 크기가 변하도록 설정한다:

```css
--text-display: clamp(3rem, 8vw, 8rem);      /* 히어로 타이틀 */
--text-h1: clamp(2.5rem, 5vw, 5rem);
--text-h2: clamp(2rem, 4vw, 3.5rem);
--text-h3: clamp(1.5rem, 3vw, 2.25rem);
--text-h4: clamp(1.25rem, 2vw, 1.75rem);
--text-body: clamp(1rem, 1.2vw, 1.25rem);
--text-small: clamp(0.875rem, 1vw, 1rem);
--text-caption: clamp(0.75rem, 0.9vw, 0.875rem);
```

### 타이포그래피 디테일

- `letter-spacing`: 대문자 텍스트에는 `0.05em~0.1em` tracking 추가
- `line-height`: 제목은 `1.1~1.2`, 본문은 `1.5~1.7`
- `font-feature-settings`: 숫자가 많으면 tabular figures 활성화
- `text-wrap: balance`: 제목에 적용하여 줄바꿈 균형 맞춤

## 스페이싱 & 레이아웃

### 스페이싱 스케일 (4px base)

```
--space-1: 0.25rem (4px)   — 아이콘과 텍스트 사이
--space-2: 0.5rem (8px)    — 인라인 요소 간격
--space-3: 0.75rem (12px)  — 관련 요소 간격
--space-4: 1rem (16px)     — 기본 간격
--space-6: 1.5rem (24px)   — 컴포넌트 내부 패딩
--space-8: 2rem (32px)     — 컴포넌트 간 간격
--space-12: 3rem (48px)    — 섹션 내 그룹 간격
--space-16: 4rem (64px)    — 소규모 섹션 간격
--space-24: 6rem (96px)    — 섹션 간격
--space-32: 8rem (128px)   — 대규모 섹션 간격
--space-48: 12rem (192px)  — 히어로/풀스크린 패딩
```

### 레이아웃 그리드

- **12컬럼 그리드**: 기본이되, 콘텐츠 영역은 8~10컬럼만 사용하여 여백을 확보한다
- **비정형 그리드**: 컨셉에 따라 2:3, 3:5 등 비균등 분할을 사용한다
- **컨테이너**: `max-width: 1440px` 기본, 풀블리드 섹션은 컨테이너를 벗어난다
- **Bento 그리드**: 가변 크기 셀로 시각적 리듬을 만든다

## 컴포넌트 스타일

### 버튼

- Primary: 배경색 채움 + 부드러운 호버 전환 (배경색 미묘하게 변화 + 약간의 scale)
- Secondary: 보더만 + 호버 시 배경 채움 애니메이션
- Ghost: 텍스트만 + 호버 시 underline 드로잉 애니메이션
- 모든 버튼에 `transition-all duration-300 ease-out` + 미세한 `active:scale-[0.98]`

### 카드

- 호버 시 그림자 깊어짐 + 미세한 y축 이동 (`hover:-translate-y-1`)
- 카드 크기를 다양하게 — 모든 카드가 같은 크기이면 AI스럽다
- 카드 내부 패딩을 넉넉하게 — 콘텐츠가 가장자리에 닿지 않게

### 네비게이션

- 스크롤 시 배경 블러 + 축소되는 sticky 헤더
- 메뉴 링크 호버: underline 드로잉 또는 배경 슬라이드 효과
- 모바일: 풀스크린 오버레이 메뉴 + 스태거 애니메이션

## 모션 시스템

### 이징 커브

```css
--ease-out-expo: cubic-bezier(0.16, 1, 0.3, 1);     /* 부드러운 감속 — 대부분의 전환 */
--ease-out-quart: cubic-bezier(0.25, 1, 0.5, 1);     /* 약간 빠른 감속 */
--ease-in-out-quart: cubic-bezier(0.76, 0, 0.24, 1); /* 대칭적 가감속 — 루프 애니메이션 */
--ease-spring: cubic-bezier(0.34, 1.56, 0.64, 1);    /* 스프링 효과 — 강조 */
```

기본 `ease-in-out`은 사용하지 않는다 — 너무 기계적이다. `ease-out-expo`가 기본 이징이다.

### 듀레이션 스케일

```
--duration-fast: 150ms     — 호버, 색상 변화
--duration-normal: 300ms   — 일반 전환
--duration-slow: 500ms     — 레이아웃 변화
--duration-slower: 700ms   — 페이지 전환
--duration-reveal: 1000ms  — 스크롤 reveal 애니메이션
```

### 애니메이션 패턴

| 패턴 | 적용 대상 | 구현 |
|------|----------|------|
| Fade Up | 스크롤로 나타나는 요소 | `opacity: 0→1, translateY: 30px→0` |
| Stagger | 리스트/그리드 아이템 | 각 아이템 `delay: index * 0.1s` |
| Clip Reveal | 이미지/텍스트 등장 | `clip-path: inset(100% 0 0 0) → inset(0)` |
| Scale In | 강조 요소 | `scale: 0.9→1, opacity: 0→1` |
| Parallax | 배경/장식 요소 | GSAP ScrollTrigger `y: -50→50` |
| Text Reveal | 히어로 텍스트 | 글자별/단어별 순차 등장 |
| Magnetic | 버튼/링크 | 커서 접근 시 미세하게 따라오는 효과 |

### 중요: 애니메이션 절제

- 모든 요소가 움직이면 아무것도 강조되지 않는다
- 핵심 콘텐츠와 CTA에 애니메이션을 집중한다
- 같은 애니메이션을 연속 사용하지 않는다 — Fade Up만 10개 반복하면 AI스럽다
- `prefers-reduced-motion` 미디어 쿼리를 반드시 존중한다

## 반응형 전략

### 브레이크포인트

```
sm: 640px    — 모바일 랜드스케이프
md: 768px    — 태블릿
lg: 1024px   — 작은 데스크탑
xl: 1280px   — 일반 데스크탑
2xl: 1536px  — 대형 모니터
```

### 반응형 원칙

- 모바일은 단순화하되 퀄리티를 유지한다 — 복잡한 그리드는 단일 컬럼으로, 하지만 타이포그래피와 여백의 품질은 유지
- 태블릿은 데스크탑의 축소가 아닌 별도 레이아웃으로 취급한다
- 터치 타겟은 최소 44x44px
- 복잡한 호버 이펙트는 터치 디바이스에서 대체 인터랙션 제공

## 최신 트렌드 반영 가이드

컨셉에 따라 적절히 선택하여 적용한다. 상세는 `references/design-trends.md` 참조.

| 트렌드 | 적합한 컨셉 |
|--------|-----------|
| Bento 그리드 | 다양한 콘텐츠를 보여줄 때 |
| Glassmorphism | 세련되고 미래적인 느낌 |
| 그라디언트 메시 | 유기적이고 부드러운 배경 |
| 키네틱 타이포그래피 | 대담하고 역동적인 사이트 |
| 다크모드 + 비비드 액센트 | 프리미엄/테크 느낌 |
| 오버사이즈 타이포그래피 | 미니멀하지만 강렬한 인상 |
| 노이즈/그레인 텍스처 | 아날로그 감성, 크래프트 느낌 |
| 스크롤 스토리텔링 | 서사가 있는 브랜드 사이트 |
| 3D 요소 | 몰입형 경험 |
